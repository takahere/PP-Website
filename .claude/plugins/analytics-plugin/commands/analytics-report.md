# アナリティクスレポート生成

レポート種別: $ARGUMENTS

## 実行手順

### 1. レポート種別の判定
- `weekly` または `week`: 過去7日間のレポート
- `monthly` または `month`: 過去30日間のレポート
- 指定なし: 過去30日間をデフォルト

### 2. データ取得
以下のエンドポイントからデータを一括取得:

```bash
# トラフィック概要
curl -s http://localhost:3000/api/analytics/ga

# 検索パフォーマンス
curl -s http://localhost:3000/api/analytics/gsc

# コンバージョン
curl -s http://localhost:3000/api/analytics/events

# チャネル別
curl -s http://localhost:3000/api/analytics/acquisition

# トレンド
curl -s http://localhost:3000/api/analytics/trends
```

### 3. レポート生成
以下のフォーマットでMarkdownレポートを生成:

```markdown
# PartnerProp ウェブサイト分析レポート
期間: YYYY-MM-DD 〜 YYYY-MM-DD

## エグゼクティブサマリー
[3-5文で主要な発見と推奨アクションを要約]

## トラフィック概要
| 指標 | 今期 | 前期比 |
|------|------|--------|
| ユーザー数 | X | +Y% |
| セッション数 | X | +Y% |
| PV数 | X | +Y% |
| 直帰率 | X% | -Y% |
| 平均セッション時間 | Xm Ys | +Y% |

## チャネル別分析
| チャネル | ユーザー | 構成比 | CVR |
|----------|---------|--------|-----|
| 自然検索 | X | Y% | Z% |
| ...

## コンバージョン分析
| イベント | 件数 | 前期比 |
|----------|------|--------|
| 資料DL | X | +Y% |
| ...

## 検索パフォーマンス（GSC）
| 指標 | 今期 | 前期比 |
|------|------|--------|
| 表示回数 | X | +Y% |
| クリック数 | X | +Y% |
| CTR | X% | +Y% |
| 平均順位 | X | -Y |

### 上位検索クエリ
| クエリ | 表示 | クリック | CTR | 順位 |
|--------|------|----------|-----|------|

## ページパフォーマンス
### 人気ページ TOP10
| ページ | PV | 滞在時間 |
|--------|-----|----------|

### 改善が必要なページ
[直帰率が高い、滞在時間が短いページ]

## 推奨アクション
1. [具体的なアクション1]
2. [具体的なアクション2]
3. [具体的なアクション3]
```

### 4. 出力
レポートを表示。ユーザーが希望すればファイルとして保存。

## レスポンスガイドライン

- **言語**: 日本語
- **数値**: 千単位はカンマ区切り、%は小数点1桁まで
- **比較**: 増加は緑（↑）、減少は赤（↓）で表現（テキストで）
- **アクション**: 具体的かつ実行可能なものを提案
