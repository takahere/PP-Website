# 期間比較分析

比較対象: $ARGUMENTS

## 実行手順

### 1. 期間の判定
- `week`: 今週 vs 先週（過去7日 vs その前の7日）
- `month`: 今月 vs 先月（過去30日 vs その前の30日）
- `YYYY-MM-DD YYYY-MM-DD`: カスタム期間比較

### 2. データ取得
GA4 APIから両期間のデータを取得:

```bash
# メインデータ（summaryに前週比が含まれる）
curl -s http://localhost:3000/api/analytics/ga

# トレンドデータ（日別推移）
curl -s http://localhost:3000/api/analytics/trends

# チャネル別
curl -s http://localhost:3000/api/analytics/acquisition

# コンバージョン
curl -s http://localhost:3000/api/analytics/events
```

### 3. 比較分析
以下の指標について比較分析を実施:

#### 基本指標
| 指標 | 期間1 | 期間2 | 変化 | 変化率 |
|------|-------|-------|------|--------|
| ユーザー数 | | | | |
| セッション数 | | | | |
| PV数 | | | | |
| 直帰率 | | | | |
| 平均セッション時間 | | | | |
| エンゲージメント率 | | | | |

#### チャネル別変化
| チャネル | 期間1 | 期間2 | 変化率 |
|----------|-------|-------|--------|
| 自然検索 | | | |
| ダイレクト | | | |
| SNS | | | |
| 参照元 | | | |

#### コンバージョン変化
| イベント | 期間1 | 期間2 | 変化率 |
|----------|-------|-------|--------|
| 資料DL | | | |
| WP DL | | | |
| デモ申込 | | | |

### 4. インサイト生成
- 最も改善した指標
- 最も悪化した指標
- 注目すべきトレンド
- 推定される要因

### 5. レポート出力

```markdown
# 期間比較分析レポート

## 比較期間
- 期間1: YYYY-MM-DD 〜 YYYY-MM-DD
- 期間2: YYYY-MM-DD 〜 YYYY-MM-DD

## サマリー
[主要な変化を3-5文で要約]

## 詳細比較
[表形式で各指標の比較]

## キーインサイト
1. **改善点**: [具体的な改善内容]
2. **注意点**: [悪化している指標]
3. **トレンド**: [継続的な傾向]

## 推奨アクション
1. [アクション1]
2. [アクション2]
```

## レスポンスガイドライン

- **変化率の表示**: +X% または -X% で明確に
- **有意性**: 大きな変化（±10%以上）は強調
- **文脈**: 季節性や外部要因も考慮
- **アクション**: 変化に基づいた具体的な提案
